Now that we've worked with a Table we created for a bit, you may be finding that working with data sets any larger than a few rows and columns will start to become more time consuming to enter in by hand. Luckily, we have a way to bypass the manual input by importing a file.

A file is just a way to store data. You may have seen files on your computer before, but their actual contents were not as important. Now, we are interested in seeing how a CSV file can help us solve our input problem. A .csv tag at the end of a filename means that the information stored in the file is separated by commas. Why or how it works is not as important, but what you do need to know is that you can import and even store a Table in this file format. When you see .csv, think Table.

The process of importing a Table in Snap! requires:

  1. A .csv file that contains Table data
  2. A global variable for you to store that data in (in this lab, we use )
  3. That global variable to be shown on the stage (by clicking the little check box next to it in the toolbar on the left of the screen)
