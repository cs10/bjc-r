Recall the code thus far for our book class, which is essentially empty: 


class Book:
    pass

It is pretty annoying to set all the attributes individually like we did in the previous page of this lab. We can add a special function to our class which will initialize (a.k.a. create) an object from a list of desired attributes. This special function is called a constructor, and in Python has the name __init__ (two underscores on each side).

An example of a constructor is given below: 


class Book:
    def __init__(self, genre, title, author, py):
        self.genre = genre
        self.title = title
        self.author = author
        self.publication_year = py

Now, when we create a Book object, we need to make sure every argument is correctly filled. In this case, we have four arguments: genre, title, author, and py. 


hunger_games = Book('Dystopian fiction', 'The Hunger Games', 'Suzanne Collins', 2008)

ranger_games = Book('Crime Nonfiction', 'Ranger Games', 'Ben Blum', 2017)

print(hunger_games.genre)
Dystopian fiction
print(ranger_games.author)
Ben Blum

As before, we can visualize the behavior of this program behavior. Do so at this link.

Key things to note:

  1. The __init__ function is automatically called when you create a Book object, i.e. x = Book(...). Furthermore, the first argument, self, is not something that we explicitly pass to the Python function. Instead, self refers to the object that is being created. This is a little weird when you frist see it as a novice programmer.
  2. Inside the code for the constructor, you'll notice that we add self in front of the name of each argument on the left hand side of each = operation. This is because we're taking the arguments passed to the function and copying their data into the current book's personal memory. For example, above, the book is handed a number called py which is equal to 2008, and the fourth line of the constructor copies this 2008 into the self's variable called "publication_year". This makes more sense if you step through the code in PythonTutor as suggested above.
  3. The four variables created in the constructor are called "instance attributes". Each instance of the Book class has its own personal data that it shares with no other Book.
