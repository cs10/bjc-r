There are two parts to a function, the "head" and the "body".

PART ONE: HEAD

To define a function in Python, you first need to make a head. It generally looks like this:

def function_name():
Try making a sum function, which will add two numbers (x and y) and output their sum. Start with a "def statement" (underlined).
def function_name():
"def" is the first thing we write when defining a function. We always write "def". It is Python's way of saying how the following function will be defined. This is analogous to button in snap

You can choose almost any function name you want. We could have used "add" or "add_two_numbers", or something else. Note the space between the "def" and the function name. That one space is essential to the syntax.

Our function will have two parameters, x and y. You can make a function with as many or as little parameters as you want. We start by opening a parenthesis following the function name. Then, name your variables, and separate them with a comma and possibly a space, although spaces are not necessary in separation. Close your parameters with a closing parenthesis.

For a function with zero parameters, just use an opening and closing pair of parentheses like this: ()

The final part of the head is the colon. It immediately follows the parameters, and distinguishes the end of your function head. You must have this colon at the end of the head. So, the final result of the head of our function will look something like this:

def sum(x, y):

PART TWO: BODY

Now we need to write the "body" of the function. Just like in snap you can fill in the body with pretty much anything: loops, conditionals (if/else), variables, even other function definitions, and more.

The most important thing to do when starting off the body is indenting (4 spaces or 1 tab).
Then, begin writing your code. Go ahead and fill in the rest of the line. So, we have something like this:


		def sum(x, y):
			return x + y
Return ends our function (as we learned previously), and outputs the value that follows. So in this case, the function returns x + y. If the function reaches a return statement, then no lines below it will be executed. The current sum function is very simple, try writing sum with a temporary variable, called total.
